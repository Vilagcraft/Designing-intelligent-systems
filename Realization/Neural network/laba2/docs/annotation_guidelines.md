# Инструкция для аннотаторов (RuReviews)

## Цель
Создать эталонную разметку для обучения модели анализа тональности и аспектов отзывов.

## 1. Классы тональности
- **positive** — отзыв явно выражает положительное отношение к товару;
- **neutral** — текст описывает факт без оценки;
- **negative** — отзыв выражает недовольство.

## 2. Аспекты (опционально)
Аннотаторы могут отмечать аспекты из списка:
`quality, price, delivery, packaging, battery, design, comfort, service`.
Указывать polarity (pos / neg / neu).

## 3. Extractive Highlights
Выделить 1–2 ключевых фрагмента, которые отражают главное мнение (по 5–30 слов).

## 4. Качество разметки
Каждый отзыв размечают два аннотатора.
Подсчитывается **Cohen’s κ**; если κ < 0.7 — пересмотреть инструкции и спорные случаи.

## 5. Не размечать
Рекламу, спам, бессмысленные тексты или тексты без мнения.

## 6. Пример
| Отзыв | Sentiment | Аспекты |
|-------|------------|---------|
| «Телефон отличный, но батарея садится быстро.» | mixed → overall negative | quality: pos, battery: neg |
